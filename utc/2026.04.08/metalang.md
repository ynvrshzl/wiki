Preface: we learn by reconstructing meaning through observed usage by other people. This means; programming should follow the same function at it's core. Constructing a system out of verbs, actions and ideas, rather than symbols and frankenstein patch-fixes (lol).  And this way, others can read our code and reconstruct meaning through observed usage. 

That is something you almost cannot fundamentally do with modern code. Just look at most github repositories; fundamentally unreadable from the outsider's perspective. And i consider this to not only be e-waste, but human-cognition-waste, becuase materals are not evergreen; they are not reuseable. They only exist to create a monolithic system. Very few systems survive that way in nature, but in software? It's normal to have giant directories where the code is only accessible by a gatekept community. It's time to change that.
