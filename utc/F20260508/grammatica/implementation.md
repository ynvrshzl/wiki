implementation is a core function of human creativity. it is the ability to, essentially, alchemically translate idea / input, into product, using a method. that is the same transformation process as the fundamental computational model: $o = x(i)$.

implementation itself is the hardest part of any code system. because, before the ability to speak in language, so to speak, one has to familiarize with the grammar, the sounds, the rules, etc. and programming is most definitely a language, so the same rules apply at the cognitive level. 

ideally, programming should simply execution as implementation. however, we often experiment, hack our way around problems, and experience constant cognitive fragmentation. this was [one of the fundamental motivations behind this project](motivation-and-backstory.md)

there are several proposed equations and symbolic layers to understand the complex stage of implementation. but as we currently understand it, implementation is not the core work of the human engineer. implementation is simply the ability to execute models.
