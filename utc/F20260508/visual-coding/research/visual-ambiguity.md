designing visual blocks is always open to interpretation, CSS makes this open and author-able. however, like with any UI/UX design, we must work in reverse, because visuals on their own are by nature in complete and required definition. 

reverse engineering from a real world lived example of navigational technological tools, Google maps for example, solves this issue by providing both literature definition, and visual navigation. of all the tools we've used, Google maps is the direct ancestral model because code is navigation.
