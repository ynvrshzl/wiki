# The Commands are the Program, The interface is the interactive layer


Recently, we checked out the [Obsidian CLI.](https://obsidian.md/cli) ...and then I realized, wait the only thing that makes obsidian more powerful than a plain text editor, is the commands! the command palette pretty much lets you access everything in the program without having to click buttons and things! and designing user interfaces as one of the hardest things for me because it's like painting on an infinite canvas. there's just way too much chaos and potential. and programming by itself is already a chaotic canvas, and so designing visual interfaces just becomes incredibly draining and consuming. but now we have re-framed the program as being commands first, and optional graphical interfaces on top. but really the CLI is completely usable as the court program itself. interface is just a more aesthetically pleasing and accessible form of the underlying program command! so then this means, that we can fundamentally reproach programming especially designing systems that require human interaction, which is most of our work. and we can approach it as commands first, graphical user interface last.
