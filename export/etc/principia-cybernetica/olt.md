Building a codex, a library or lexicon of universal computational laws, systems, definitions, patterns, and essentially, building foundational understanding on how to utilize computers properly.

it's actually incredible that software engineering is one of the only fields that does not standardize the construction at substrate level; it merely measures the tangible metrics at the scale of production; the measurable effects of software, hardware, and its derivative observable properties.

however, the software engineer operates primarily between two domains, the substrate, and the effect. in essence, a software engineer works in an environment where cause and effect can be authored, and where unpredictability could cause system instability, errors are predicted. systems are strengthened. as a result, a third intersection appears, the sphere of security. of system integrity. this is synonymous with civil engineering concept of structural integrity, the reliability of a structure for sustaining life without collapse. 



From the phenomenological, epistemological, ontological, psychological perspective; software is not just producing result, the way an engineering tool is crafted for a specific function; software seems to be a phenomenon; it is both
- a tooling of function (e.g., code that is processed by the computer, as a product.)
- a function of tooling (e.g., code that is reasoned by the human, as an interface language.)

I could almost describe software as, physically embedded, non-physical virtual tool chains. but the philosophical, ontological studies of computation, are geared towards the ethical implications, of which, a whole section of a separate work is dedicated, for which I plan we carefully study and Orient these principles properly. 

here is the core of the study; phenomenology of software engineering. why is authoring code so difficult? why is it so simple to make statements, but it is so difficult to build systems? what other fields dealt with this? what solutions did they implement? what is software missing? is software a unique virtual phenomenon? or is it symbolic, pairing it closer to mathematics then engineering? but in that sense, software is not merely computation; software is an interface language, between multiple domains of technologies, mediums, hardware, environments, runtime. 

I do believe it is essential to define what software really is, today, not from the isolated inspections of mathematicians or data scientists, but how software is actively changing how we live our lives as human beings, happy local and collective scale!
