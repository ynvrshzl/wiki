# Ideas for "Author's Syntax"
The purose of the Author's Syntax is twofold; to convey ideas as plaintext, and to provide markdown-inspired syntax for an [AuthorIDE](#)

| Symbol  | Function    | Description | Examples | 
| ---     | ---     | ---         | --- | 
| `[...]` | Boundary   | Describes a body of text as placeholder. This is proposed solution to [writer's block](). For this purpose, the sequence of characters is titled a "Ghost block." | This happened so long ago becuase[...]
| `(...)` | Passage   | Describes a passage
| `<...>` | Block   | A system of code...
| `{x}`   | Block   | Similarly to the "Ghost block", the variable describes a semantic aspect of the literature that is missing.  | e.g., "The person went {x} becuase of {y}." |
| `{a, b, c}` | Branch | Branching versions of in-line edit
| `@edt` | Tag | A semantic tag for editing
