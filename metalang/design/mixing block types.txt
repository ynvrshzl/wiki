--- Mixing block types ---
A fundamental idea in programming, which we explored in "Concepts/" is the idea of multiple responsibilities/domains within a block of program.

That's actually what makes programming so powerful, so we don't want to remove it, but we do want to make it visible, that we are switching domains of thought. We are never in two domains at once, that's possible for computers, but impossible for a human brain.

--- Real-world scenario ---

For example, if we want to apply mathematics, conditions and error checking...

--- Classical programming ---

function min(x, y){
	if (x < y) return x;
    else return y;
};

min(32, 48); // returns 32

--- Proposed solution ---

Transformer("min").inputs(32, 48).calculate()

new Transformer("min").inputs("x", "y")
    .switch("logic")
    	.return("x").if("x").islessthan("y")
    	.otherwise().return("y")
	.switch("output")
    	.keyword("calculate")
    	.format("console")
