In the computational evolution of systems, Queries are historically significant becuase they help us solve a significant problem; "How do we navigate a system, that was once linear & manageable, and has emerged into non-linear chaos?" The answer is; Structured inquiry. It is essentially the architecture of information management.

There are two fundamental models of a query in a computation system: The query expression/prompt, and the view.

- A query is an "inquiry" of a system, using properties and logic to "ask" questions about the system e.g., "where file.name = todo". This is an example of both: logic and property. We are accessing a property (which is structured information, data) about each member of a system, using logical expression to operationalize information as expressions.
- A view is a saved state of a query, such that the structure of a query remains constant, while the information changes in the system.
