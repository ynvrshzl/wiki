essentially a class can be thought of as two ideas, instantiation and blueprint. what we are doing with meta Lang is combining both of them into the actual class. this means that we are replacing classical programming with classes and objects. the reason we do this is because it is the only way to read a program like we read human language, and then the same way, this is how we express the program as we created. this undoubtedly leads to less errors and frustration, which is the ethical somatic purpose of this project.
