# Structures for information transmission
Conceptualizing structures as dimensions of information, helps establish the idea that we are essentially "mapping information space, as a cognitive equivalent of a spatial context."

In this sense, this is the "Cartography of information for human beings."

## 1D, link 
uni-dimensional, bi-directional.

A link is a __uni-dimensional__ information object, as it can only represent one location in any given space, e.g., "An html \<link\> can only show one string at a time." 

However, a link is bi-directional, in the sense that, while it forms a uni-dimensional point, it creates an edge between systems. 

e.g.,

"a" links to "b" as a one-way edge. "b" can link to "a" creating a mutual edge.

## 2D, Property
two-dimensional, quad-directional

A property is a semantically defined interface for an information object. We can conceptualize the property as being a two-dimensional information interface structure of a link...

e.g.,

```markdown
"title" and "key" creates a "property" for "a". 
```

Now, the information object "a" has a property "title" and "key", which can be referenced, queried, calculated, and structured further, by nesting properties with data structures.
