# What is software used for?

### Is coding difficult?
At it's most fundamental level, software is all about solutions & methods. You have an idea for a solution, to a problem. But how do you actually make it real? **The answer is translation.** Translating the idea into software, by learning the method (language of software). This means we need to first learn the language of software. This allows us to create a dictionary of programming methods, similar to a poke-dex! For example, If I want to make my site customizable, I can first meditate/think of solutions that will be useful. Perhaps my site should have settings, and changeable parameters. This is where the translation comes in. From here, I research online, the language and it's documentation on how to do these things. This encourages me to think in terms of the language (methods) eventually, seamlessly creating a bridge between the idea world and the machine.

### Always start basic
**Remember that the more things you add, the more problems you'll have to solve.** At each level, make sure that you can see the big picture, without being overwhelmed by details. If you're overwhlmeed by the amount of things going on, it's probably time to rebuild from scratch! Remember, starting over doesnt mean starting from zero.

What is software? What is the purpose of software? Software can better the life of the planet.

essentially software is just a language between humans and machines. in order for the language to be useful, there have to be layers between the machine and the human. human beings can understand abstract and logical words, but a machine can only understand numbers. almost all of the effort of software comes from translating those numbers back up to our human language, and allowing our language to translate back to numbers. it has to be a two-way system in order for it input and output to be mutual.

but software is unlike any other kind of engineering, because physical engineering deals with the physical world, but software engineering, exists in a nearly infinite canvas. software is abstracted, not only because there are layers of abstractions that make it useful, but software itself is abstract. we are creating with structures that don't exist in real life, and we solve problems in software that only exist in the computer world. 

none of this would be important to me if I didn't see the power of technology in the right hands. for example in medical applications surgical robots only exist because of computers and languages have been developed. these robots have saved countless lives. AI is also starting to be used properly after decades of misuse. 

one of my dreams that keeps me alive is that someday human beings will learn to use technology to solve problems in nature and not create new ones.
