IF the purpose of programming languages are to help human beings think in terms of computational models, then we need a language that fundamentally is a translator between -> human expression -> and -> computation models.

fundamentally, this could be achieved by the fundamental constructs of a language itself: in the models. the way we think about the world, about problems and solutions. 

Research on Human cognitive constraints:

Constraint 1: Expression axis
How do we human beings actually express things?
We can think in terms of multiple dimensions, connecting thoughts together like nodes on a network, but fundamentally we can only travel in one axis at a time.

Computers, can handle multiple dimensions at once. This explains why code can become so difficult; fundamentally the order of execution is not the same as the order in which we express ideas.

In mathematics, we specifically describe the order in which things should happen. But in programming? This is usually seen as "a super power" but for human beings, this is what makes code fundamentally difficult to work with and specially read. That leads us to the second constraint.

Constraint 2: Expression vs Reading
When we express ideas in code, it is easy to get carried away. We forget, often, that we'll have to come back to the program, and read it in order to understand what the program is doing.

Fundamentally, if we make the expressibility of code as natural as humanly possible, perhaps the retrieval of information is less cognitively taxing, and less fucking headaches!

Solutions:
Constraint 1, is represented in metalang, as a fundamental design construct -> code reads <-> left to right, ^-v top down. just like natural language. The order in which things are written, should be the order of execution.
