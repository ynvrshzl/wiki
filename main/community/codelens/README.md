# Codelens

## 🚀 Compressed description of this project

A system that turns source code into a a queryable, navigable map, combining structure, narrative, and visualization—without replacing the medium of plain-text code as source.

## Twofold design
This project contains 2 core ideas

1. CQL (Code Query Language)
2. Code visualized as maps (using cql)

What's the point of writing code for code? At a low level, it seems like extra work, but in a complex system, something like this is built into the most secure IDE's (e.g. JetBrains IDEs, Microsoft Visual Studio, etc.)

## 🧭 Code as a Map

Code should remain plain text as the source of truth. A visual + narrative overlay on top, helps us conceptualize code like terrain, not just text, which is **the only sustainable way to manage complex computational systems.** Infact, it is the only way we human beings manage complex fields.



---

🧠 Key Principles

1. **Representation over Replacement:** Don’t replace programming languages

2. **Navigation orientation:** Change how we cognitively perceive and navigate source code systems.
3. **Multi-Layer System:** Narrative + Visual layers. (a) Narrative layer... provides literature guidance, which is how we human beings establish meaning in a ambiguous space (maps are geometric ambiguity until verbally explained.) Comments, docs, meaning. (b) Visual layer... structure as a map


4. **Comments Become First-Class.** Documentation lives inside the code. Not separate websites or files. Acts as narrative guidance, not just notes. This is, the natural intuition...


---

# 🗺️ The Map Analogy

Source code is ambiguous terrain. Comments are annotations, landmarks, and explaining the meaning behind structure.

# Goal
> The goal: navigate code like exploring a world, not reading a scroll


# ⚙️ Design

✅ This IS:

- A visualization layer
- A perception tool
- A code navigation system

❌ This is NOT:

- A new programming language
- Traditional visual programming (nodes as source)
- A replacement for text-based coding


---

# 🔍 Core Features (Prototype-Level)

Parse existing code (AST-based)

Source code now has object-oriented-esque properties, where keywords, data, and more and parsed into the AST. This is essentially the meta-source of the code, which we can operate, query, and visualize into meaningful visuals to solve human engineer cognitive issues.

---

# 🔬 Critical Design Constraint

Zoom Levels, This follows the mechanics of a visual map, in which only relevant information

Zoom out → architecture

Zoom in → functions

Zoom deeper → implementation


> Prevents visual overload / “node explosion”




---

# 🧬 Philosophy

Programming is akin to executable literature.

Humans think in: stories, space, relationships

Current systems don't even allow that domain of thought, we're researching how to make it the core activity behind human software engineering.

Sustainable systems should: Allow messy human expression first, refine into precise computation afterward.
