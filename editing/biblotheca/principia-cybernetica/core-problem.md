The core problem in all of computer science, isn't discovering even technological, it's a much harder problem: __Understanding how our tools affect us, and change the way we think.__

This is fundamentally a feedback system.

Arguably, coding isn't hard because it's difficult to learn; Code is hard because it does not respect human cognition. 

We think in terms of 
- Space
- Navigation
- Chunks
- Roles
- Properties
- Identity 
- Relationships

Code forces us to think in
- Static execution
- Non-linear narrative, expressed in linear space. (This is how literature works, and how natural language works too.)

Code is structure does not reflect the phase it was created in.

For example, take this principle coding model:

1. The engineer begins with an idea, solution or vision.
2. The engineer explores possibility first, writig messy but _locally meaningful_ code.
3. Eventually, the engineer begins abstracting, when it feels right. 
4. The abstractions are built on unstable understanding.
5. Now the system is: Neither exploratory (open, flexible, extensible) nor architectural (clear, contained, structured)

This is the worst possible state: **Fragmented proto-structure.**
