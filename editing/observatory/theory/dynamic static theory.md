# Dynamic-Static Theory (DST)

**Aliases:** Source vs. Implementation

## Abstract

**Abstract** - This theory analyzes how complex computational systems (...and possibly other types of systems...) are comprised of this fundamental model; Dynamic and static. An interwoven relationship between static sources of truth, and dynamic interpretations. 

- **Static** describes the source medium, of which data or information is transmissible. Data and information continously evolve, while the medium itself is static.
- **Dynamic** describes the model interface, displaying and retrieving and reading from static sources of information.

## Practical applications
The reason this model is important, is because personally we've observed that in a knowledge Management system, something that appears to be primitive and simple, actually explodes in complexity. rather than analyzing the complexity emergence, it is more useful to analyze models which make the emergent chaos, manageable by human beings. 

Amongst the various models we were observing, it seems that the static dynamic model, reduces human input, but not human agency. that is really the Core essence of Chaos agency. and so this model is a fundamental part of our studies. 

so imagine we have a static source of data, plain text files. to make content readable, we start formatting and the blinking things, but as the number of files and information increases, formatting becomes inconsistent and rigorous and a design problem. and so we keep the information as static and irreducible as possible, while keeping the dynamic elements like formatting and properties, open to interpretation through dynamic representation. 

so essentially this is where the power of queries comes. being able to query data, not only retrieves information, but you also have control over how to display that information, which is the Core essence of [UI/UX](ui-ux).

other applications of this system, are also conceptual. and conceptual schematics are applicable to many forms and mediums. so this model, fundamentally describes how the most sustainable systems that deal with interfacing and displaying information, fundamentally are comprised of static units of information, and dynamic representations. this means that the data, is a source of truth, rather than the medium it exists in. and when the source of truth is irreducible, the design itself becomes assistive and adaptive and accessible, rather than the sole focus of technology. and perhaps, most of the vision in engineering systems, is actually lost ironically in the vision of the interface. this might be related to the concept of program as commands, in which designing a program, fundamentally follows a model of commands first, interface last. and when we think in terms of this reverse engineering, this is actually how the engineer should approach constructing applications. the graphical user interface never comes before the actual program. 

and the program never comes before the irreducible static source of truth. 

but the system is not monolithic, it's not perfect, and it's only a model that best describes how to work with systems that display information. such as websites, applications, knowledge management, etc.
